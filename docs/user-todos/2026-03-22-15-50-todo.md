# Frontend And Guild Status

This file started as a raw idea dump. It now tracks what from that note has shipped and what is still intentionally deferred.

## Shipped from the earlier guild note

- Authenticated users now have a dedicated `/guild` home.
- Guild masters are redirected to `/guild` until first-time setup is completed.
- Guild settings now include a saved guild time zone.
- Access control based on guild rank is implemented for:
  - creating `GUILD` raids
  - signing up to `GUILD` raids
- Public raids remain available to any logged-in user.
- Site admins have an explicit `/guild/admin` override flow instead of silent elevated access.

## Still deferred from the earlier guild note

- guild branding beyond the current guild identity data:
  - mirrored crest rendering
  - guild slogan
- using guild MOTD as the signed-in landing page

## Other frontend ideas still open

- normalize signup button sizing and one-click state transitions
- split raid signups into clearer "in" and "out" groupings
- improve desktop raid layout density without regressing mobile
- replace the old header logo
- expand character cards with portrait-rich paging/grid behavior
- run the separate performance review on response times, transfer size, and cost/performance
